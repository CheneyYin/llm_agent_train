# 演示脚本 03.7：Promptfoo — 提示词测试

## 场景准备

```bash
# 确保 promptfoo 已安装
npm install -g promptfoo

# 进入项目目录
cd demo-project
```

> **运行环境**：Windows（CMD 或 PowerShell），命令已适配。Linux/macOS 用户命令相同。

---

### 步骤 1：为什么测试提示词

**前情过渡**：
"Tool 给 Agent 装上了手。但还有一个根本问题——提示词本身的质量怎么保证？"

**讲师话术**：
"你写了一个提示词，改了改，怎么知道改对了？看一次输出就觉得没问题？"

"我们写代码不会靠'看起来没问题'就上线——写好测试，跑通，提交。提示词也一样。promptfoo 就是把 TDD 带入提示词工程的工具。"

**关键展示点**：
- 类比：提示词测试 = 单元测试 for LLM
- 核心问题：没有客观标准时，优化提示词等于瞎调

**衔出过渡**：
"测试需要三个要素——被测的东西、谁跑、什么算对。promptfoo 把这三件事组织成一个 YAML 文件。"

---

### 步骤 2：promptfoo 核心概念

**前情过渡**：
"测试需要三个要素。"

**讲师话术**（打开 `scripts/promptfooconfig.yaml`）：

"看这个文件——三个核心概念："

| 概念 | YAML 字段 | 说明 |
|------|----------|------|
| prompts | `prompts:` | 要测试的提示词，可以写多个版本 |
| providers | `providers:` | 用哪个模型跑 |
| tests | `tests:` | 测试用例 + 断言 |

"我准备了两个提示词版本——版本 A 直接问，版本 B 加了角色设定和 JSON 输出约束。3 个测试用例，每个用例有一段 log 内容 + 断言。"

**展示**（讲师指向 `promptfooconfig.yaml` 逐段讲解）：

- 版本 A："分析以下日志中的错误：{{log_content}}" — 没有角色，没有格式约束
- 版本 B："你是资深 SRE。分析以下日志，以 JSON 数组格式输出……" — 有角色 + 结构化输出
- 断言：`contains-json`、`JSON.parse(output).length >= 2`、`icontains: "null value"`

**展示 Mermaid 流程图**（取自讲义）：写提示词 → 定义测试 → eval → 查看 → 改进 → 循环

**关键展示点**：
- YAML 声明式配置，一行命令跑全部测试
- 支持 90+ 模型提供商
- 支持 30+ 断言类型

**衔出过渡**：
"概念讲完了，直接跑一遍。"

---

### 步骤 3：实战演示

**前情过渡**：
"概念讲完了，直接跑。"

**讲师话术**：
"现在我在终端跑 `promptfoo eval`——它会用 GPT-4o 分别跑两个版本的提示词，逐个测试用例验证断言。"

**命令**：
```bash
promptfoo eval
```

**预期输出**：
- 命令行显示测试进度
- 版本 A（无角色）通过率低——输出格式不统一，不满足 JSON 断言
- 版本 B（有角色+结构化）通过率 2/2 或接近

**讲师话术**：
"看——版本 A 的 JSON 断言没过，因为提示词没有要求 JSON 输出。版本 B 全过了。这不是靠'感觉'，是数据告诉你的。"

**命令**：
```bash
promptfoo view
```

**预期效果**（讲师在浏览器中展示 Web UI）：
- 并排对比：版本 A 输出 vs 版本 B 输出
- 通过/失败标记（绿色 ✓ / 红色 ✗）
- 延迟对比（版本 B 稍慢因为提示词更长？）
- Token 成本对比

**关键展示点**：
- 有数据就不靠感觉——哪个版本好，一目了然
- 每次改提示词，重新跑 `promptfoo eval`，立即知道改对了还是改错了
- CI/CD 集成：可以把 `promptfoo eval` 放进 GitHub Actions，提示词改动自动回归测试

**备用方案**：
- 若 Live API 调用太慢（> 2 分钟），讲师提前跑好并保存结果，现场用预先打开的 Web UI 展示
- 若 promptfoo 版本不兼容，改为展示 YAML 配置 + 口头讲解概念

**衔出过渡**：
"现在提示词可以测试了，Tool 可以调用了。把这两者组装起来——可测试的提示词 + 可调用的工具 + 自动化的思考循环——这就是 Agent。"
